1 Declaration Types at a Glance
ENS

Entry Summary Declaration

ENS000… / DEC000…
Safety & security declaration submitted to HMRC via ICS2 before goods move to NI. Usually completed by the carrier/haulier. Carrier, Buyer, and Seller must use XI or EU EORIs.
When Required
All GB → NI movements via RoRo and air into inventory-linked locations. TSS does not support ENS for maritime or RoW → NI.
SFD

Simplified Frontier Declaration

SFD000… / DEC000…
Auto-generated by TSS from the ENS information when a consignment is submitted. Submitted to CDS before goods move, using a simplified dataset. Non-controlled goods are held as EIDR instead of a full SFD.
When Created
Automatically when an ENS consignment is submitted under the TSS Simplified Procedure. TSS submits it on your behalf.
SDI

Supplementary Declaration

SUP000…
Full customs declaration completed after goods arrive in NI — closes the customs journey. The Importer EORI on the SFD determines who completes it. TSS auto-generates a draft SDI which the trader updates and submits.
Deadline
Must be submitted by the 10th calendar day of the month following the goods' arrival. Periodic Declarations can consolidate multiple SDIs.
FFD

Full Frontier Declaration

FFD000…
All customs info in one declaration. Required for Rest of World (excl. EU) → NI and Inventory-Linked Ports. No information carries over from the ENS — must be completed separately. Duties calculated on submission.
Declaration Categories
H1 (free circulation), H2 (warehousing), H3 (temporary admission), H4 (inward processing).
IMMI

Internal Market Movement Information

GLR000…
Simplified declaration for "not at risk" goods moving GB → NI. Requires UKIMS authorisation and XI EORI. Available as pre-movement (IMD, ENS-triggered) or post-movement (IMZ, SDI conversion).
When to Use
When the importer holds UKIMS and goods will stay in NI/UK — won't enter the EU single market.
GMR

Goods Movement Reference

GMR000… → GVMS ID
Links the physical vehicle/transport to the customs declarations. Required for every commercial vehicle moving GB → NI. Combines ENS MRN + SFD MRN (or EIDR) pairs. Generates a barcode for the driver.
Created After
All consignments are "Authorised for Movement". TSS can auto-create or traders can build via GVMS API.

4 Movement Corridors — Which Declarations Apply

🚛 GB → NI (RoRo, Standard)

TSS Simplified Procedure — most common PLE journey
ENS SFD (auto) GMR SDI (after arrival)

🚛 GB → NI (Not at Risk, UKIMS)

Internal Market Movement — simplified for qualifying traders
ENS IMMI (pre-mvmt) GMR

✈️ GB → NI (Air, Inventory-Linked)

Full Frontier required for inventory-linked locations
ENS FFD

🌍 RoW (excl. EU) → NI

ENS not via TSS — Full Frontier only
FFD (arrived)

🚢 GB → NI (Maritime)

Pre-lodged or arrived FFD — TSS does not support maritime ENS/SFD
FFD (pre-lodged or arrived)

📦 Special Procedures (H2/H3/H4)

Warehousing, Temporary Admission, Inward Processing
FFD (H2/H3/H4) ENS (if RoRo GB→NI)
5 Not at Risk vs At Risk — Duty Impact

✅ Not at Risk

Goods staying in NI/UK — won't enter EU single market. Zero or reduced duty GB → NI. UKIMS holders use simplified IMMI process.

AI Codes: NIDOM (domestic GB→NI), NIREM (not at risk, UKIMS required)

⚠️ At Risk

Goods that could enter the EU. EU tariff rates apply — duties must be paid. Full Supplementary Declaration or FFD required.

AI Code: NIIMP (UK and EU control measures applied)
Subsidy Waiver: NIAID (offsets duty against allowance)

7 Step-by-Step Process Flows

A TSS Simplified Procedure (GB → NI RoRo) — PLE Primary Flow

PRE-MOVEMENT
1. Create Declaration Header
POST declaration_headers (op_type: create)
Returns ENS000… reference. Contains movement/transport info.
2. Create Consignment(s)
POST consignments (op_type: create)
Returns DEC000…. Links to ENS header. Importer, exporter, goods desc.
3. Create Goods Items (1–99)
POST goods (op_type: create)
Returns hex ID. Commodity code, weights, procedure code, packages.
4. Submit Consignment
POST consignments (op_type: submit)
TSS creates ENS → HMRC. Auto-generates SFD (or EIDR for non-controlled). Header locked after 1st submit.
5. Create & Submit GMR
POST gvms_gmr (op_type: create/submit)
After all consignments "Authorised for Movement". Combines ENS + SFD/EIDR MRN pairs. Generates GVMS barcode.
POST-MOVEMENT
6. Lookup Supplementary Declaration
GET supplementary_declarations?sfd_number=DEC…
Returns SUP000… reference. TSS auto-drafts from SFD data.
7. Update SDI + Goods Items
POST supplementary_declarations (op_type: update)
POST goods (op_type: update, consignment_number: SUP…)
Add full customs info: valuation, additional procedures, documents.
8. Submit SDI
POST supplementary_declarations (op_type: submit)
Deadline: 10th of month following arrival. Closes the customs journey. Duties calculated.

B Full Frontier Declaration (RoW → NI / Inventory-Linked / Special Procedures)

1. Create FFD
POST full_frontier_declarations (op_type: create)
Returns FFD000…. Specify declaration_category (H1–H4), movement_type, carrier, importer, arrival details.
2. Add Goods Items (1–99)
POST goods (op_type: create, consignment_number: FFD…)
Full commodity detail, duty info, documents, licences. Different mandatory fields per H-type.
3. Submit FFD
POST full_frontier_declarations (op_type: submit)
Sent to CDS. Duties calculated on submission. If rejected, update and re-submit.
4. Read FFD (get MRN / approval)
GET full_frontier_declarations?reference=FFD…&fields=status,mrn
For pre-lodged declarations, FFD + ENS MRNs sent to GVMS → GMR returned.
Note: No information carries across from the ENS to the FFD — the FFD is a completely separate declaration.

C Internal Market Movement (IMMI) — Not at Risk, UKIMS

PRE-MOVEMENT (IMD) — ENS Triggered
1–4. Same as Simplified Procedure
Create Header → Consignment → Goods → Submit. TSS detects UKIMS and generates IMMI instead of SFD/SDI path.
5. Update IMMI if rejected
POST internal_market_movement (op_type: update, submit)
Optional — only if HMRC rejects the IMMI submission.
POST-MOVEMENT (IMZ) — Standalone
1. Create standalone IMMI
POST internal_market_movement (op_type: create, declaration_category: IMZ)
Returns GLR000…. arrival_date_time must be in the past. Requires UKIMS-associated EORI.
2. Add Goods Items + Submit
POST goods → POST internal_market_movement (submit)
Simplified goods info. free_circulation_home_use must be "Yes" for standard IMMI.
8 Declaration Status Lifecycles
DeclarationStatus FlowKey Trigger
ENS Consignment Draft → Submitted → Processing → Authorised for Movement Consignment submit call
SFD Draft → Submitted → Processing → Authorised for Movement → Arrived Auto-created on ENS submit. If rejected: Amendment Required → re-submit
SDI (Sup Dec) Draft → Submitted → Processing → Accepted / Rejected Trader submits after arrival (deadline: 10th of following month)
FFD Draft → Submitted → Processing → Accepted / Rejected Trader submits. If rejected: update goods/details, re-submit
IMMI Draft → Submitted → Processing → Accepted / Rejected ENS-triggered (IMD) or standalone (IMZ)
GMR Draft → Open → Checked In → Embarked → Complete Submit after all consignments authorised. GVMS assigns gmr_id.
