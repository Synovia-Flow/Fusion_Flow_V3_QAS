Who Provides What — Actor Roles in TSS Declarations

Each field in a TSS declaration is owned or provided by a specific actor in the supply chain. Understanding ownership drives automation: fields from master data can be auto-populated; fields that only the haulier knows must come from the booking feed; HMRC/TSS-generated fields are read-back only.

Master Data vs Derived vs Transactional

Knowing what is master data (held once, reused) versus derived (calculable from master + booking input) versus transactional (unique per movement) is fundamental to pipeline automation. Master data lives in CFG / BKD schemas and is pre-populated. Transactional data must be captured per crossing or consignment.

Master Data — held once, reused
Derived — calculable from master + input
Transactional — unique per movement / consignment
Auto-generated — issued by TSS / HMRC
