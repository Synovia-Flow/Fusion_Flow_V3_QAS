ENS Entry Summary Declaration

Prefix: ENS000... / DEC000...
Safety and security declaration submitted before goods move to NI via ICS2. Usually completed by the carrier/haulier. Carrier, Buyer, and Seller EORIs must be XI or EU prefixed. The ENS is the starting point for both the Simplified Procedure and FFD journeys.
DB: [CWF].[EnsHeaders], [CWF].[EnsConsignments], [CWF].[EnsGoodsItems]
Process Steps
#StepAPI CallDetail
1Create Declaration HeaderPOST /declaration_headers {op_type: "create"}Returns ENS000... ref. Set movement_type, arrival_port, identity_no_of_transport, vehicle_reg_num, route.
2Update Header (optional)POST /declaration_headers {op_type: "update", ens_lrn: "ENS..."}Add/change transport details. Not available once 1st consignment submitted.
3Create Consignment(s)POST /consignments {op_type: "create", ens_lrn: "ENS..."}Returns DEC000... Set importer_eori, exporter_eori, consignor, consignee, goods_description, controlled_goods, transport_document_number.
4Create Goods Items (1-99)POST /goods {op_type: "create", consignment_number: "DEC..."}Returns hex goods_id. Set commodity_code, gross_weight_kg, procedure_code, type_of_package, number_of_packages.
5Submit ConsignmentPOST /consignments {op_type: "submit", consignment_number: "DEC..."}Triggers ENS to HMRC + auto-creates SFD (or EIDR). Header locks after 1st submit.
6Read Consignment StatusGET /consignments?reference=DEC...&fields=status,error_messagePoll until "Authorised for Movement". If "Trader Input Required" check error_message and fix.

SFD Simplified Frontier Declaration

Prefix: SFD000... / DEC000...
Auto-generated by TSS from ENS consignment data when submitted. Sent to CDS before goods move. Non-controlled goods held as EIDR. Controlled goods get full SFD with MRN. Traders cannot create SFDs — only update/re-submit if rejected.
DB: simplified_frontier_declaration, sfd_header
Process Steps
#StepAPI CallDetail
1Auto-Created on ENS Submit(automatic — no API call)TSS creates SFD from ENS consignment. Controlled → full SFD. Non-controlled → EIDR.
2Lookup SFD NumberGET /simplified_frontier_declarations?consignment_number=DEC...Returns sfd_number.
3Read SFD Status & MRNGET /simplified_frontier_declarations?reference=DEC...&fields=status,movement_reference_number,eori_for_eidrCheck Authorised for Movement. Get MRN or eori_for_eidr.
4Update SFD if RejectedPOST /simplified_frontier_declarations {op_type: "update", sfd_number: "DEC..."}Only if status = 'Amendment Required'. Fix goods_description, holder_of_authorisation, etc.
5Update SFD Goods ItemsPOST /goods {op_type: "update/create/delete", consignment_number: "DEC..."}Goods cloned from ENS. Can add, update, delete on SFD.
6Re-Submit SFDPOST /simplified_frontier_declarations {op_type: "submit", sfd_number: "DEC..."}Re-submit after fixing rejection.

SDI Supplementary Declaration

Prefix: SUP000...
Full customs declaration completed after goods arrive in NI. Closes the customs journey. TSS auto-generates a draft from SFD data — trader updates with valuation, procedures, documents, then submits. Deadline: 10th calendar day of the month following arrival.
DB: supplementary_declaration
Process Steps
#StepAPI CallDetail
1Lookup SDI from SFDGET /supplementary_declarations?sfd_number=DEC...Returns SUP000... May return multiple for periodic declarations.
2Read SDI (pre-populated)GET /supplementary_declarations?reference=SUP...&fields=status,importer_eori,...Check what needs updating.
3Update SDI ConsignmentPOST /supplementary_declarations {op_type: "update", sup_dec_number: "SUP..."}Add valuation_method, freight_charge, insurance, guarantees.
4Update/Add SDI Goods ItemsPOST /goods {op_type: "update", consignment_number: "SUP...", goods_id: "hex..."}commodity_code, item_invoice_amount, additional_procedure_code, documents.
5Submit SDIPOST /supplementary_declarations {op_type: "submit", sup_dec_number: "SUP..."}Deadline: 10th of following month. CDS calculates duties.
6Read Duty LinesGET /duty?sup_dec_number=SUP...duty, duty_vat, duty_excise, duty_total per goods item.
7Handle Rejection / ReclassifyPOST {op_type: "update"} + re-submit OR {op_type: "reclassify"} to IMMIFix and re-submit, or reclassify H1→H8 if goods qualify for IMMI.

FFD Full Frontier Declaration

Prefix: FFD000...
All customs info in one declaration. For RoW (excl. EU) → NI, Inventory-Linked Ports, special procedures (H2-H4). No info carries from ENS. Supports H1-H4 categories. Duties calculated on submission.
DB: full_frontier_declaration
Process Steps
#StepAPI CallDetail
1Create FFDPOST /full_frontier_declarations {op_type: "create", declaration_category: "H1"}Returns FFD000... Set movement_type, arrival, carrier, importer, location_of_goods.
2Update FFD (optional)POST /full_frontier_declarations {op_type: "update", ffd_number: "FFD..."}Add/change header. All fields re-sent on update.
3Add Goods Items (1-99)POST /goods {op_type: "create", consignment_number: "FFD..."}Full detail. Mandatory fields vary by H1-H4.
4Submit FFDPOST /full_frontier_declarations {op_type: "submit", ffd_number: "FFD..."}Sent to CDS. Duties calculated. MRN sent to GVMS if pre-lodged.
5Read FFD Status + MRNGET /full_frontier_declarations?reference=FFD...&fields=status,movement_reference_numberPoll until accepted. MRN on success.
6Handle RejectionUpdate FFD/goods + re-submitRead error_message, fix, re-submit.
7Read Duty LinesGET /duty?ffd_number=FFD...Duty breakdown per goods item.

IMMI Internal Market Movement Information

Prefix: GLR000...
Simplified declaration for "not at risk" goods GB → NI. Requires UKIMS authorisation + XI EORI. Pre-movement (IMD, ENS-triggered) or post-movement (IMZ, standalone). Can be created by reclassifying SDI H1→H8.
DB: internal_market_movement
Process Steps
#StepAPI CallDetail
1aPre-Movement (IMD) — ENS Triggered(auto — via ENS submit if UKIMS)TSS creates IMMI instead of SFD/SDI path.
1bPost-Movement (IMZ) — StandalonePOST /internal_market_movement {op_type: "create", declaration_category: "IMZ"}Returns GLR000... arrival_date_time must be past. internal_market: "confirmed".
2Add Goods ItemsPOST /goods {op_type: "create", consignment_number: "GLR..."}Simplified goods. commodity_code, gross_weight, country_of_origin.
3Submit IMMIPOST /internal_market_movement {op_type: "submit", glr_number: "GLR..."}Must be Draft or Trader Input Required.
4Read Status / MRNGET /internal_market_movement?reference=GLR...&fields=status,movement_reference_number,immi_confirmation_datePoll until Authorised for Movement.
5Reclassify SDI→IMMIPOST /supplementary_declarations {op_type: "reclassify", sup_dec_number: "SUP..."}H1→H8. Returns new GLR000...
6Reclassify IMMI→SDIPOST /internal_market_movement {op_type: "reclassify", glr_number: "GLR..."}H8→H1. Returns SUP000...

GMR Goods Movement Reference

Prefix: GMR000... → GVMS gmr_id
Links physical vehicle/transport to customs declarations. Required for every commercial vehicle GB → NI. Combines ENS + SFD/EIDR MRN pairs. GVMS allocates gmr_id and barcode for driver.
DB: gvms_gmr
Process Steps
#StepAPI CallDetail
1Create GMRPOST /gvms_gmr {op_type: "create", ens_lrn: "ENS..."}Set route_id, vehicle_reg_num, trailer_registration_nums.
2Update GMR (if transport changes)POST /gvms_gmr {op_type: "update", gmr_lrn: "GMR..."}Must re-submit after update.
3Submit GMRPOST /gvms_gmr {op_type: "submit", gmr_lrn: "GMR..."}Wait 90s for gmr_id. Status → 'Open'. Barcode generated.
4Read GMR + GVMS IDGET /gvms_gmr?reference=GMR...&fields=gmr_id,gmr_status,inspection_requiredGet gmr_id, check inspection_required.
5Cancel GMRPOST /gvms_gmr {op_type: "cancel", gmr_lrn: "GMR..."}Only if Draft/Open/TIR. Cancelling Open deletes from GVMS.
